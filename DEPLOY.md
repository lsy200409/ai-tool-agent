## 1. 系统要求

- **浏览器**：Google Chrome 88+（Manifest V3 要求）
- **操作系统**：Windows / macOS / Linux
- **Node.js**：16+（本地工具服务器需要）
- **网络**：能访问目标 AI 平台（DeepSeek、ChatGPT 等）

## 2. 快速安装（开发者模式）

这是目前最简单的安装方式，无需注册开发者账号。

### 2.1 获取扩展包

下载 `ai-tool-agent-v3.0.0.zip` 并解压到任意目录。

### 2.2 加载扩展

1. 打开 Chrome，地址栏输入 `chrome://extensions/`
2. 右上角开启 **「开发者模式」** 开关
3. 点击 **「加载已解压的扩展程序」**
4. 选择解压后的文件夹（包含 manifest.json 的目录）
5. 扩展加载成功，工具栏出现 AI Tool Agent 图标

### 2.3 验证安装

- 访问 https://chat.deepseek.com ，页面右侧应出现悬浮宠物图标
- 点击宠物图标可展开/收起工具面板
- 面板显示「已连接」表示工具服务器通信正常

## 3. 本地工具服务器部署

扩展需要本地工具服务器来执行实际操作（文件读写、命令执行等）。

### 3.1 安装依赖

```bash
cd deepseek-tool-agent
npm install
```

### 3.2 启动服务器

**方式一：直接启动**

```bash
npm run server
```

**方式二：使用启动脚本（Windows）**

双击 `start-server.bat` 或在 PowerShell 中运行：

```powershell
.\scripts\start-server.bat
```

**方式三：开机自启动（可选）**

```powershell
# 以管理员身份运行
powershell -ExecutionPolicy Bypass -File .\scripts\setup-autostart.ps1
```

### 3.3 验证服务器

服务器默认运行在 `http://localhost:3002`，启动后应显示：

```
[ToolServer] 工具服务器已启动 → http://localhost:3002
[ToolServer] 已注册 X 个工具
```

## 4. 使用流程

### 4.1 基本使用步骤

1. **启动工具服务器**（见第 3 节）
2. **打开 AI 平台**（如 chat.deepseek.com）
3. **注入工具提示词**：按快捷键 `Ctrl+Shift+I`（或点击面板中的「注入提示词」按钮）
4. **正常对话**：在聊天框中输入需要工具协助的请求
5. **确认工具执行**：当 AI 请求工具调用时，弹出确认对话框，点击「确认」执行
6. **查看结果**：工具执行结果自动回填到对话中，AI 基于结果继续回复

### 4.2 工具面板说明

- **宠物图标**：页面右侧悬浮，点击展开/收起面板
- **日志区域**：显示工具调用的完整执行过程
- **连接状态**：显示与本地工具服务器的连接状态
- **快捷操作**：提供常用工具的快速入口

### 4.3 安全确认机制

所有工具执行前都会弹出确认对话框，显示：
- 工具名称
- 执行参数
- 「确认」和「拒绝」按钮

只有用户点击「确认」后才会执行，拒绝则跳过该工具调用。

## 6. 配置说明

### 6.1 工具服务器地址

默认：`http://localhost:3002`

如需修改，编辑 `src/core/config.js` 中的 `SERVER_URL`。

### 6.2 工具配置

工具配置文件位于 `workspace/config/tools_config.json`，可自定义：
- 启用/禁用特定工具
- 设置工具参数默认值
- 配置安全限制（如文件访问路径白名单）

### 6.3 插件系统

插件目录：`workspace/plugins/`

每个插件包含：
- `index.js` — 插件逻辑
- `plugin.json` — 插件元数据

将新插件放入此目录，重启工具服务器即可加载。

### 6.4 技能系统

技能目录：`workspace/skills/`

技能通过 SKILL.md 文件定义，提供更复杂的工具编排能力。

## 7. 常见问题排查

### Q: 面板不显示 / 宠物图标不出现

1. 确认扩展已启用：`chrome://extensions/` 中检查
2. 刷新 AI 平台页面（F5）
3. 检查控制台是否有错误（F12 → Console）

### Q: 工具服务器连接失败

1. 确认服务器已启动：`npm run server`
2. 检查端口 3002 是否被占用
3. 检查防火墙是否阻止了本地连接

### Q: AI 不调用工具

1. 确认已按 `Ctrl+Shift+I` 注入工具提示词
2. 确认对话中明确需要工具协助（如"帮我查看当前目录"）
3. 检查面板日志中是否有「工具提示词已注入」记录

### Q: 工具执行后无结果

1. 检查面板日志中的执行状态
2. 确认工具服务器正在运行
3. 查看工具服务器终端的错误输出

### Q: 确认对话框按钮无法点击

1. 刷新页面重试
2. 确认扩展版本为最新

## 8. 发布到 Chrome 插件商店

> **为什么别人的插件可以从商店下载？**
> 因为他们完成了 Chrome Web Store 的开发者注册和审核发布流程。目前我们的插件以开发者模式加载，功能完全相同，只是安装方式不同。发布到商店后，用户就可以像安装其他插件一样一键安装。

### 8.1 前置条件

- **Google 开发者账号**：一次性注册费 **5 美元**
- **扩展 ZIP 包**：已生成的 `ai-tool-agent-v3.0.0.zip`
- **隐私政策页面**：需要一个公开可访问的隐私政策 URL
- **推广素材**：截图、宣传图等

### 8.2 注册开发者账号

1. 访问 [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole)
2. 使用 Google 账号登录
3. 支付 5 美元一次性注册费（信用卡/Google Pay）
4. 填写开发者信息（姓名、邮箱等）
5. 同意开发者协议
6. 完成邮箱验证

### 8.3 准备商店素材

| 素材 | 要求 | 说明 |
|------|------|------|
| 图标 | 16×16 / 48×48 / 128×128 | 已有 |
| 截图 | 1280×800 或 640×400 | 1-5 张，展示核心功能 |
| 宣传图 | 440×280 | 可选，商店详情页展示 |
| 详细描述 | 最长 16000 字 | 功能介绍、使用说明 |
| 简短描述 | 最长 132 字 | 一句话概括 |
| 隐私政策 | 公开 URL | 说明数据收集和使用方式 |

### 8.4 提交审核

1. 登录开发者控制台，点击 **「新建项目」**
2. 上传 ZIP 包
3. 填写商店信息（名称、描述、截图、分类等）
4. 填写隐私政策 URL
5. 说明每个权限的用途：
   - `activeTab`：与当前标签页内容交互
   - `storage`：保存用户设置
   - `tabs`：管理标签页切换
   - `nativeMessaging`：与本地工具服务器通信
   - `alarms`：定时任务调度
6. 设置可见性（公开/未列出/私密）
7. 点击 **「提交审核」**

### 8.5 审核时间

- 普通审核：1-3 个工作日
- 权限较多的扩展：可能延长至 7 天
- 首次提交可能稍长

### 8.6 常见拒绝原因

1. **权限请求不合理** — 确保每个权限都有明确用途
2. **隐私政策缺失** — 必须提供公开的隐私政策页面
3. **功能描述与实际不符** — 描述需准确反映功能
4. **包含调试代码** — 已清理 console.log
5. **截图质量差** — 使用高清截图，展示真实界面

### 8.7 审核通过后

- 扩展立即在商店上线
- 用户可搜索并一键安装
- 后续更新只需上传新版本 ZIP，自动推送给用户

### 8.8 替代分发方式

如果暂时不想支付 5 美元注册费，还有以下方式分发：

- **开发者模式加载**（当前方式）：功能完整，需手动操作
- **企业内部分发**：通过 Google Workspace 管理员推送，仅限组织内用户
- **提供 CRX 文件下载**：`chrome://extensions/` 中点击「打包扩展程序」生成 .crx 文件（Chrome 已限制安装商店外的 CRX，不推荐）

## 9. 发布到 Edge 插件商店

Edge 基于 Chromium，完全兼容 Chrome 扩展，**无需修改任何代码**即可使用和发布。

### 9.1 Edge 开发者模式安装

1. 打开 `edge://extensions/`
2. 开启 **「开发人员模式」**
3. 点击 **「加载解压缩的扩展」** → 选择解压后的文件夹
4. 扩展加载成功，功能与 Chrome 版完全一致

### 9.2 Edge 插件商店发布（免费）

> **Edge 商店注册免费！** 不像 Chrome 要 5 美元，微软不收取开发者注册费。

#### 注册开发者账号

1. 访问 [Microsoft Partner Center](https://partner.microsoft.com/dashboard/microsoftedge/public/login?ref=addons)
2. 使用微软账号登录
3. 填写开发者信息并提交
4. 等待邮箱验证通过（通常 1-2 天）

#### 提交扩展

1. 登录 Partner Center，点击 **「创建新扩展」**
2. 上传 ZIP 包（与 Chrome 版相同的 `ai-tool-agent-v3.0.0.zip`）
3. 设置可见性：公开 / 隐藏（仅链接可访问）
4. 选择分发市场（默认全球）
5. 填写扩展属性：分类、隐私政策 URL、支持联系方式
6. 填写商店信息：
   - 描述（250-10000 字符）
   - 商店 Logo（300×300 像素）
   - 截图（640×480 或 1280×800，最多 10 张）
   - 搜索关键词
7. 添加认证说明（可选，帮助测试人员理解扩展功能）
8. 点击 **「发布」** 提交审核

#### 审核时间

- 通常 1-7 个工作日
- 首次提交可能稍长
- 审核通过后立即上线
