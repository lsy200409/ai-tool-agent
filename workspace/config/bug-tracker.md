##  BUG 列表

### BUG-001: 插件管理工具在对话中不可用
- **发现时间**: 2026-06-15
- **严重程度**: 高
- **现象**: `plugin_list`、`plugin_install`、`plugin_uninstall`、`plugin_reload` 在服务器端已注册（日志显示"已注册 5 个插件管理工具"），但在对话中调用时返回"未知工具"
- **影响**: AI 无法自主扩展能力，无法安装新插件
- **原因分析**: 可能与 system prompt 中的工具列表同步机制有关——这些工具通过 `registerPluginManagerTools()` 注册，但未出现在前端注入的工具提示词中
- **建议修复**: 检查 tool-server.js 的 `/api/tools` 或 `/exec` 端点是否完整返回了插件管理工具，确认前端 tool-registry 同步逻辑

### BUG-002: 技能管理工具在对话中不可用
- **发现时间**: 2026-06-15
- **严重程度**: 高
- **现象**: `skill_list`、`skill_create`、`skill_toggle` 同样不可用
- **原因分析**: 与 BUG-001 同源，技能管理工具也通过服务器端注册但未同步到前端对话可用工具列表
- **建议修复**: 与 BUG-001 一起排查工具同步机制

### BUG-003: exec_command 在 WSL 模式下中文输出乱码
- **发现时间**: 2026-06-15 (日志 10:41:08)
- **严重程度**: 中
- **现象**: WSL 中执行 `ls -la ~/xiao-zheng-skills` 返回的 stdout 包含乱码字符（如 `�Spe\u0019��`）
- **影响**: AI 无法准确读取 WSL 命令的输出结果，导致后续判断错误
- **建议修复**: 检查 exec_command 在 WSL 模式下的编码处理，确保 stdout 以 UTF-8 解码

### BUG-004: 用户消息重复记录
- **发现时间**: 2026-06-15
- **严重程度**: 低
- **现象**: 日志中大量出现同一用户消息被记录两次（如 ` 用户消息(XX字): "..."` 出现连续两条）
- **原因分析**: SSE 模式 + DOM 轮询模式同时触发导致重复事件
- **建议修复**: 在 input-monitor.js 中增加去重逻辑

### BUG-005: exec_command 在 Windows 非 WSL 模式下间歇性 ENOENT
- **发现时间**: 2026-06-15
- **严重程度**: 中
- **现象**: 执行 `dir /b /s` 等命令时出现 `spawn C:\WINDOWS\system32\cmd.exe ENOENT`
- **原因分析**: 可能是进程 spawn 时的路径解析问题或权限问题
- **建议修复**: 检查 server/cross-platform.js 中的 spawn 逻辑，增加路径回退机制

##  优化建议

### OPT-001: 工具同步机制需要改进
- 当前服务器注册了 33 个工具，但前端注入提示词和对话可用工具列表可能不一致
- 建议：服务器提供一个 `/api/tools/full` 端点，返回完整的已注册工具列表（包括插件管理工具和技能管理工具）

### OPT-002: 技能热加载
- 当前通过 `write_file` 在 workspace/skills/ 写入 SKILL.md 后，需要重启服务器或重新注入才能生效
- 建议：实现技能目录的文件监听，新技能写入后自动触发 `skill_list` 刷新

### OPT-003: WSL 命令执行的中文支持
- 当前 WSL 模式下的中文输出存在编码问题
- 建议：在 exec_command 的 WSL 执行路径中显式设置 `LANG=zh_CN.UTF-8` 或类似环境变量

### OPT-004: 文件写入的安全性日志
- 当前 `write_file` 和 `append_file` 的日志仅记录到 tool-server 日志
- 建议：对工作区外的写入尝试增加告警日志，并在 UI 面板中展示最近的文件操作记录

### OPT-005: 技能系统与前端 UI 的集成
- 当前技能列表通过面板注入，但缺少可视化的技能浏览器
- 建议：在 UI 面板中增加"技能"标签页，显示已安装技能列表及启用/禁用状态
