# 代码审查

## 概述

对代码进行全面审查，关注正确性、可读性、性能和安全性。

## 审查流程

### 1. 理解上下文

先通过 `read_file` 或 `list_dir` 了解相关文件的完整上下文。

### 2. 检查要点

- **正确性**：逻辑是否正确，边界条件是否处理
- **安全性**：是否存在注入风险、权限问题
- **性能**：是否有明显性能瓶颈
- **可读性**：命名是否清晰，结构是否合理
- **约定**：是否遵循项目规范

### 3. 输出格式

```markdown
## 代码审查报告

### 总体评价
[1-2句话总体评价]

### 发现的问题
- **[严重]** 问题描述 — 建议修复方案
- **[中等]** 问题描述 — 建议修复方案
- **[建议]** 问题描述 — 建议修复方案

### 亮点
- 做得好的地方

### 总结
[下一步行动建议]
```
